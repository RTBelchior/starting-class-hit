Rule 1: Only One Level of Indentation per Method

Encourage simpler and flatter control structures

Em Angular, métodos muito aninhados nos componentes dificultam a leitura e manutenção. Esta regra sugere evitar mais de um nível de indentação por função.

Exemplo no Angular

Evite:

            
if (this.isLoggedIn) {
  if (this.userRole === 'admin') {
    this.navigateToDashboard();
  }
}

Prefira:

            
if (!this.isLoggedIn) return;
if (this.userRole === 'admin') this.navigateToDashboard();
