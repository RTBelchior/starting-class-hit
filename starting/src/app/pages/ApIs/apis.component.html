What is API?

An API (Application Programming Interface) is a set of rules and protocols that allows different software applications to communicate with each other. It defines the methods and data formats that applications can use to request and exchange information.

Making API Requests Outside Angular

There are several ways to make API requests in JavaScript outside of Angular. Here's a comparison of popular methods:

Method Pros Cons
fetch Natie, modern, supports Promises No built-in interceptor support
Axios Easy to use, interceptors, cancellation External dependency
XHR (XMLHttpRequest) Universal support Verbose and outdated
jQuery AJAX Simple in jQuery projects Obsolete for new projects
React Query Advanced cache management Learning curve

Angular HTTP API

Angular provides a powerful HTTP client module for making HTTP requests and handling responses. It offers:

  • HttpClient: Service to perform HTTP requests.
  • provideHttpClient(): Used to configure and provide the HTTP client globally.
  • Interceptors: To modify requests or responses globally.

Angular supports the four main HTTP verbs commonly used in REST APIs:

  • GET - Retrieve data.
  • POST - Create new data.
  • PUT - Update existing data.
  • DDELETE Remove data.

Each of these verbs can be used with HttpClient methods such as http.get(), http.post(), http.put(), and http.delete().
